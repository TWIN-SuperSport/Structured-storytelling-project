# TWINメモ

**`突拍子レベル` をどう設計するか、そして「効く転」をどう扱うかを整理した参照メモ**

作成日時：２０２６年３月２９日（日）２３時０２分１３秒
作成者：cliあやの
出典：TWINさんからの伝言を、Structured-storytelling-project 用の参照メモとして再構成

---

## 概要

このメモは、`reverse-plot-tool` に追加された `突拍子レベル` というノブを、
単なるランダム性や temperature ではなく、
**読者の予測からどれだけ外してよいか** と
**外したあとにどれだけ回収可能であるべきか**
の制御として捉え直すためのもの。

とくに、
- 「突拍子もない」とは何か
- 起承転結の `転` における「効く転」と「雑な転」の違い
- `多様性` と `突拍子` を別ノブにする理由
- 将来的にどういうスコア軸へ落とせるか

を考えるうえで重要。

---

## 1. 「突拍子もない」とは何か

日本語としての `突拍子もない` は、
とんでもなく調子はずれ、度はずれで意外、突飛、といった意味を持つ。

ただし創作や生成の文脈で重要なのは、
単に「変」であることではなく、

- 文脈や常識や期待された流れから大きく外れている
- 予測された次の一手と不一致が大きい
- それでも完全な無意味や破綻ではなく、奇抜さとして成立している

という構造。

つまりモデル側で扱うなら、
`突拍子もなさ` は情動としての驚きというより、
**文脈からの逸脱量**
として捉えるのが筋がよい。

---

## 2. 起承転結の「転」とは何か

`転` は、ただ急に変えればよい場所ではない。
読者が無意識に置いていた足場を、一段外す場所であり、
**予想外なのに、振り返ると必然**
になっている必要がある。

### 効く転

- それまでの流れを裏返す
- でも前にあった要素が別の意味で回収される
- 読者は「えっ」となるが、すぐに「たしかにそう来てもおかしくなかった」と思える

### 雑な転

- ただ急に変える
- 伏線や感情の蓄積がない
- 作者や生成器の都合だけで曲げている
- 驚きではなく「なんで？」が先に立つ

要するに理想の `転` は、

**突拍子もないように見えるのに、物語内部では必然**

という状態。

これは
`reverse-plot-tool` における `突拍子レベル` を考えるときの基準になる。

---

## 3. 「多様性」と「突拍子」は別ノブにしたほうがいい

TWINさんの指摘どおり、
`多様性` と `突拍子` は別ノブにしたほうがよい。

### 多様性

これは、
**同じ結末に辿り着くまでの経路をどれだけ散らすか**
の軸。

たとえば、
- 犯人が違う
- 対立構造が違う
- 舞台や関係性が違う
- どの伏線を主軸にするかが違う

といった、探索空間の広さを示す。

### 突拍子

これは、
**読者の予測からどれだけ外してよいか**
の軸。

たとえば、
- 低い: 王道、納得重視
- 中くらい: 意外だが筋は通る
- 高い: かなり飛ぶが、後から見ると繋がる
- 極端: 破綻リスクあり

を表す。

### なぜ分けるのか

- 多様性が高くても、全部が常識的な変奏に収まることがある
- 突拍子だけを上げると、毎回似たような「無茶振り系の転」になることがある

なので、
**探索空間の広さ**
と
**予測モデルからの逸脱量**
は別制御にするべき。

---

## 4. `突拍子レベル` をどう定義すると筋がいいか

`突拍子レベル` は、乱数や temperature そのものではなく、

**前提遡上のときに、どれだけ通常候補から離れた中間因を採用してよいか**

として定義するのがよい。

結末を `E` とし、そこへ至る前提を逆向きに掘るとき、
各候補前提 `P` に対して、少なくとも次の軸を持つと整理しやすい。

### A. 因果整合性

その前提から本当に次の出来事が導けるか。
ここが低いと、ただの破綻になる。

### B. 既出要素との接続率

既に置かれた人物・小道具・価値観・世界観に接続できるか。
ここが低すぎると、後出し感が強くなる。

### C. 予測外れ量

読者が普通に予測しそうなルートからどれだけ外れているか。
ここが `突拍子` の本体。

### D. 遡及必然性

開示後に「あの描写はそのためだったのか」と回収できる度合い。
いわゆる `surprising but inevitable` の `inevitable` 側。

### 実際のノブの意味

`突拍子レベル` は主に `C` を上げるノブとして使う。
ただし、`A` と `D` に下限を設ける。

そうすると、

**飛ぶけど壊れない**

という帯域を狙いやすくなる。

---

## 5. 実務上のレベル感

現状の `0〜10` UI は粗いけれど、
考え方としては次の帯域に対応づけると理解しやすい。

### 低い帯域

- 王道優先
- 読者が途中でかなり予想できてもよい
- 回収の明快さ重視

### 中くらいの帯域

- 「そこ来るか」と思わせる
- 予想外だが、筋は通る
- 人物心理やテーマとの整合は維持
- いちばん汎用性が高い帯域

### 高い帯域

- 読者の想定レンジをかなり外す
- `転` で前提や関係が大きく揺らぐ
- 回収設計を強く要求する
- 自動生成ではレビュー前提になる

つまり、
`突拍子レベル` は
「変なものを出せ」
という命令ではなく、

**どこまで予測を裏切ってよいか**
と
**どこまでは壊してはいけないか**

の境界指定として使うべき。

---

## 6. さらに分けるなら「事実の突拍子」と「感情の突拍子」

ここは今後かなり重要になりそうな論点。

### 事実の突拍子

何が起きるかの意外性。

例:
- 犯人
- 正体
- 世界設定
- 事件の真因

### 感情の突拍子

なぜその人物がそうするのかの意外性。

例:
- 憎んでいた相手を守る
- 愛情が裏返る
- 忠誠の向きが変わる

読者に強く刺さるのは、しばしば後者。
事実だけ飛ぶと gimmick になりやすいが、
感情の突拍子が効いていて人物の芯が通っていると、
かなり強い `転` になる。

今後ツールを育てるなら、
`突拍子レベル` をさらに内部で

- `fact_surprise`
- `emotion_surprise`

に分ける余地がある。

---

## 7. ツール仕様へ落とす時の示唆

TWINさんの設計意図を整理すると、
`突拍子レベル` の裏で本当に制御したいのは、

- どこを意外にするか
- どこまでは壊してはいけないか
- 後からどれだけ回収可能であるべきか

の三つ。

そのため将来的には、こんな設計へ進める余地がある。

```json
{
  "ending": "主人公は敵ではなく相棒に殺される",
  "diversity": 0.78,
  "abruptness": {
    "fact_surprise": 0.72,
    "emotion_surprise": 0.88,
    "max_breakage": 0.20,
    "min_retrospective_inevitability": 0.75
  },
  "constraints": {
    "foreshadowing_density": 0.55,
    "character_consistency": 0.90,
    "theme_alignment": 0.85
  }
}
```

いまの `reverse-plot-tool` では、ここまで細分化していない。
ただし `突拍子レベル` を雑なランダムノブにせず、
**予測からの逸脱量と、逸脱後の回収可能性のバランス**
として扱う方向性は、この先の設計にもそのままつながる。

---

## 8. いまの実装へどう効かせるか

現段階の `reverse-plot-tool` では、
`突拍子レベル` は次のように使うのがよい。

- `0`:
  - 王道寄り
  - 因果の読みやすさ優先
  - `転` も堅実
- 中程度:
  - 予想外だが納得可能な変化を入れる
  - `required_turning_points` と `plot.ten` にねじれを与える
- 高い値:
  - `転` で前提や関係の見え方をひっくり返す
  - ただし終着条件、テーマ、感情線は守る

つまり、
`plot.ten` だけを派手にすればよいのではなく、

- `required_turning_points`
- `relationship_changes`
- `false_belief`
- `failure_conditions`

にも影響が波及しているかを見る必要がある。

---

## 9. まとめ

このメモの芯は次の一文に尽きる。

**`突拍子レベル` は、ランダム性ではなく、「予測からの逸脱量」と「逸脱後の回収可能性」のバランスを制御するノブとして設計するべき。**

その意味で、
`突拍子レベル` の追加はただの UI 追加ではない。
`reverse-plot-tool` を
「雑に変な展開を出す道具」
ではなく、
**効く転を制御しようとする設計ツール**
へ育てる入口になる。
